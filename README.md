# 配置文件
> 在`Git` or `GitHub`中某些文件拥有特殊的作用

### `.gitignore`
> 这个文件的作用就是告诉Git哪些文件不需要添加到版本管理中

### `.gitattributes`
> 告诉Github，咱们项目是啥类型

### `.gitkeep`
> Git会忽略空的文件夹, gitkeep可以允许空文件夹

### `.babelrc`
> babel的配置文件

### `.eslintrc`
> ESLint的配置文件，支持JSON和YAML两种语法

### `.eslintignore`
> 和.gitignore文件一样，都是用来说明需要忽略的文件。

### `.github`文件夹
> 可定义一些`GitHub`的样式 或 个性化

### `.editorconfig`
> 定义不同编辑器相同的规范

### `.travis.yml`
> Travis Ci的配置说明文件
